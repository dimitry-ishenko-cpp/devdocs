Note

This httpapi plugin is part of the cisco.mso collection (version 2.9.0).

You might already have this collection installed if you are using the ansible package. It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install cisco.mso.

To use it in a playbook, specify: cisco.mso.mso.

New in cisco.mso 1.2.0

Synopsis

  • This MSO plugin provides the HTTPAPI transport methods needed to initiate a connection to MSO, send API requests and process the response.

Parameters

Parameter

Comments

login_domain

string

The login domain name to use for authentication.

The default value is Local.

Configuration:

  • Environment variable: ANSIBLE_HTTPAPI_LOGIN_DOMAIN
  • Variable: ansible_httpapi_login_domain

Hint

Configuration entries for each entry type have a low to high priority order. For example, a variable that is lower in the list will override a variable that is higher up.
