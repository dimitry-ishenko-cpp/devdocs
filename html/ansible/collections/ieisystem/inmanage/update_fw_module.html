ieisystem.inmanage.update_fw module – Update firmware

Note

This module is part of the ieisystem.inmanage collection (version 2.0.0).

You might already have this collection installed if you are using the ansible package. It is not included in ansible-core. To check whether it is installed, run ansible-galaxy collection list.

To install it, use: ansible-galaxy collection install ieisystem.inmanage. You need further requirements to be able to use this module, see Requirements for details.

To use it in a playbook, specify: ieisystem.inmanage.update_fw.

New in ieisystem.inmanage 1.0.0

Synopsis

  • Update firmware on ieisystem Server.

Requirements

The below requirements are needed on the host that executes this module.

  • Python 3.7+
  • inManage

Parameters

Parameter

Comments

dual_image

string

Update dual image(default) or not.

Only the M5 model supports this parameter.

Choices:

  • "single"
  • "dual" ← (default)

Only the M5 model supports this parameter.

over_ride

integer

Reserve Configurations, 0-reserve, 1-override.

Examples

- name: Update fw test
  hosts: inmanage
  connection: local
  gather_facts: false
  vars:
    inmanage:
      host: "{{ ansible_ssh_host }}"
      username: "{{ username }}"
      password: "{{ password }}"

  tasks:

  - name: "update bios"
    ieisystem.inmanage.update_fw:
      url: "/home/wbs/SA5112M5_BIOS_4.1.8_Standard_20200117.bin"
      type: "BIOS"
      provider: "{{ inmanage }}"

  - name: "update bmc"
    ieisystem.inmanage.update_fw:
      url: "/home/wbs/SA5112M5_BMC_4.17.7_Standard_20200430"
      mode: "Auto"
      type: "BMC"
      dual_image: "dual"
      provider: "{{ inmanage }}"
