ReQL command: close

Command syntax

conn.close([boolean])

Description

Close an open connection.

Closing a connection normally waits until all outstanding requests have finished and then frees any open resources associated with the connection. By passing false as the boolean argument to close, the connection will be closed immediately, possibly aborting any outstanding noreply writes.

A noreply query is executed by using the runNoReply command, indicating that the command should not wait for the query to complete before returning. You may also explicitly wait for a noreply query to complete by using the noreplyWait command.

Example: Close an open connection, waiting for noreply writes to finish.

conn.close();

Example: Close an open connection immediately.

conn.close(false);
