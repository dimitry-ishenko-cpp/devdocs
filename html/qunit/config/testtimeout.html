QUnit.config.testTimeout

version added: 1.0.0

Description

Set a global default timeout in milliseconds after which a test will fail. This helps to detect async tests that are broken, and prevents tests from running indefinitely.

type number or undefined
default undefined

This can be overridden on a per-test basis via assert.timeout(). If you donâ€™t have per-test overrides, it is recommended to set this to a relatively high value (e.g. 30000 for 30 seconds) to avoid intermittent test failures from unrelated delays one might in a browser or CI service.
