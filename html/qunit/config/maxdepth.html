QUnit.config.maxDepth

Categories: QUnit.config.

version added: 1.16.0.

Description

In the HTML Reporter, the depth up-to which an object will be serialized during the diff of an assertion failure.

type number
default 5

To disable the depth limit and allow infinite depth, use a value of 0.

This is used by QUnit.dump.parse().

Changelog

UNRELEASED Make QUnit.dump.maxDepth an alias for QUnit.config.maxDepth, allowing both to be read and changed at runtime.
QUnit 1.18 Introduce QUnit.config.maxDepth to enable setting via preconfig. Temporary changes at runtime must change QUnit.dump.maxDepth instead.
QUnit 1.16 Introduce QUnit.dump.maxDepth.
