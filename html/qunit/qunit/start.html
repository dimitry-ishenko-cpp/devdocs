Description

QUnit.start()

Start the test runner manually, when QUnit.config.autostart is false. For example, if you load test files with AMD, RequireJS, or ESM dynamic imports.

Note: See QUnit.config.autostart for detailed examples of how to use this.

Warning: Prior to QUnit 1.16, this method was used for resuming an async QUnit.start function, as complement to QUnit.stop(). To resume asynchronous tests, use assert.async() instead.
